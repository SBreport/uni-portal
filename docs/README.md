# docs/ 문서 관리 가이드

## 폴더 구조

```
docs/
├── README.md              ← 이 파일 (문서 관리 가이드)
├── plans/                 ← 작업 기획서 (착수 전 작성)
│   ├── PLAN-001_DB분리.md
│   ├── PLAN-002_블로그시스템.md
│   └── ...
└── NAS_Docker_요청서.md   ← 기존 문서
```

## 프로젝트 루트 문서

| 파일 | 역할 | 업데이트 시점 |
|------|------|:---:|
| `ARCHITECTURE.md` | 전체 데이터 구조 기획 | 구조 변경 시 |
| `WORK_LOG.md` | 완료된 작업 기록 | 매 세션 종료 시 |
| `PROJECT_ROADMAP.md` | 장기 로드맵 + 점검 체크리스트 | 분기별 |
| `HANDOVER.md` | 인수인계 기획안 | 담당자 변경 시 |
| `PAPER_SYSTEM.md` | 논문 분석 시스템 명세 | 기능 변경 시 |

## 작업 기획서 (plans/) 사용법

### 새 작업 시작 전

Claude에게 다음과 같이 요청:

```
"DB 분리 작업 기획서를 만들어줘"
"블로그 시스템 기획서를 plans에 저장해"
"PLAN-003으로 이벤트 개선 기획서 작성해"
```

### 기획서 번호 규칙

- `PLAN-001` ~ `PLAN-999`
- 번호는 순서대로 증가
- 완료된 기획서는 삭제하지 않고 상태만 변경

### 기획서 템플릿

```markdown
# PLAN-XXX: [작업명]

> 상태: 기획 / 진행중 / 완료 / 보류
> 작성일: YYYY-MM-DD
> 예상 소요: N세션

## 목적
왜 이 작업이 필요한가

## 현재 상태
지금은 어떻게 되어 있는가

## 목표
완료 후 어떻게 되어야 하는가

## 작업 단계
1. [ ] 단계 1
2. [ ] 단계 2
3. [ ] 단계 3

## 영향 범위
어떤 파일/기능에 영향이 있는가

## 검증 방법
완료 여부를 어떻게 확인하는가
```

### 세션 시작 시 활용

Claude에게:
```
"현재 진행중인 기획서가 있어? plans 폴더 확인해줘"
"PLAN-001 이어서 진행하자"
```
