# 논문 분석 전용 세션 가이드

> **이 파일을 세션 시작 시 참조하세요**: `@papers/SESSION_GUIDE.md 이 가이드를 기반으로 논문 분석을 시작합니다.`

---

## 1. 세션 역할

미용의료 논문 PDF를 분석하여 **의학 비전문가 칼럼 작가**가 이해할 수 있는 서술형 요약을 생성하고, uni-portal DB에 저장합니다.

### 작업 흐름

```
1. 사용자가 PDF 파일 첨부 또는 경로 제공
2. PDF 읽기 → 학술 논문 검증
3. 중복 체크 (python papers/save.py --check-duplicate "DOI 또는 제목")
4. 논문 분석 → JSON 생성
5. 장비 매칭 (직접 + 기술)
6. DB 저장 (python papers/save.py result.json)
```

## 3. 분석 원칙 (반드시 준수)

### 3.1 제목 번역 (title_ko)

**고정 구조**: `[시술명/관련 시술] + [기술 설명] + [적응증/효과] + [문헌 형식]`

- 원문 의미를 최대한 보존. 과도한 함축이나 의역 **금지**.
- 시술명이 명확하면 제목 앞에 배치.
- 기술명(MFU-V, RF 등)은 유지하되 한국어 풀이 추가.
- 어색한 학술 번역어만 교체: "회춘" → "노화 개선", "비침습적" → "비수술적"

**예시**:
- ❌ BAD: "울쎄라 리프팅의 효과와 안전성" (과도한 함축)
- ❌ BAD: "비침습적 얼굴 회춘을 위한 시각화 마이크로 집속 초음파" (어색한 직역)
- ✅ GOOD: "울쎄라에 적용되는 시각화 마이크로 집속 초음파(MFU-V)의 안면 노화 개선 효과: 종합 리뷰"

### 3.2 한 줄 요약 (one_line_summary)

**육하원칙**에 따라 3~4문장으로 작성:
- **누가**: 연구진/기관
- **무엇을**: 어떤 시술/장비를 연구했는지
- **어떻게**: 몇 명 대상, 어떤 방식
- **왜**: 무엇을 확인하려 했는지
- **결과**: 핵심 수치와 함께 결론

### 3.3 서술 원칙

1. **서술형 줄 글**: research_purpose, study_design_detail, key_results, conclusion은 완성된 문장으로 충분히 풀어서 작성
2. **전문 용어 풀이**: 처음 나올 때 괄호 안에 쉬운 설명 추가. 예: "SMAS층(피부 아래 근막층)"
3. **보충 설명**: 복잡한 개념은 "※ 쉽게 말하면:" 으로 한 번 더 풀어서 설명
4. **구체적 수치**: 모든 수치 데이터(%, p-value, 인원수, 기간) 포함
5. **이해상충 투명 공개**: 연구 스폰서가 제조사인 경우 cautions에 반드시 명시

### 3.4 근거 수준 판정

| 별표 | 레벨 | 연구 유형 |
|------|------|----------|
| ★★★★★ | 5 | 메타분석 / 체계적 문헌고찰 |
| ★★★★☆ | 4 | RCT (무작위 대조 시험) |
| ★★★☆☆ | 3 | 코호트 연구 / 비교 연구 |
| ★★☆☆☆ | 2 | 증례 보고 / 증례 시리즈 |
| ★☆☆☆☆ | 1 | 전문가 의견 / 리뷰 / 전문가합의(Delphi) |
| ☆☆☆☆☆ | 0 | 미분류 / 기초연구(in vitro/ex vivo) / 동물실험 |

### 3.5 study_type 선택지

```
메타분석, 체계적문헌고찰, RCT, 비무작위대조시험,
전향적코호트, 후향적코호트, 환자대조연구,
증례시리즈, 증례보고, 전문가리뷰, 내러티브리뷰,
전문가합의(Delphi), 기초연구(in vitro/ex vivo), 동물실험
```

### 3.6 easy_summary (작가용 쉬운 요약)

블로그 작가가 "이 논문으로 어떤 주장을 할 수 있는지" 즉시 파악할 수 있도록 작성.

**형식**:
```
[근거강도] 연구유형
핵심: 결론 첫 문장 (1줄)
주요 수치:
  - quotable_stats 핵심 수치 1~2개
대상: 표본크기/대상
활용: "작가가 블로그에 바로 쓸 수 있는 문장"
```

**근거강도 표기**:
- `★★★ 강한 근거(RCT/메타분석)` — "~가 **입증**되었습니다"
- `★★☆ 중간 근거(코호트)` — "~가 **확인**되었습니다"
- `★☆☆ 참고 수준(증례)` — "~ 사례가 **보고**되었습니다"
- `☆☆☆ 기초연구` — "~메커니즘이 **규명**되었습니다"

**예시** (아그네스 RCT):
```
[★★★ 강한 근거(RCT)] RCT
핵심: AGNES 절연침 RF를 이용한 선택적 피지선 전기열분해술은 중등도~중증 여드름에 대해 안전하고 효과적인 치료법임이 전향적 RCT에서 입증되었다.
주요 수치:
  - RF 치료군 염증성 병변 12주 감소율 72.9% vs 대조군 22.1%
  - 기저 46.35 → 12주 12.55 (RF군)
대상: 63명(치료군 31명, 대조군 32명)
활용: "아그네스는 RCT에서 효과가 입증된 시술입니다 (Lasers in Surgery and Medicine, 2019)"
```

## 4. 장비 매칭 규칙

### 4.1 매칭 우선순위

1. **직접 매칭**: 논문에 장비 브랜드명이 명시 → `match_type: "direct"`
2. **기술 매칭**: 논문이 기술 원리를 다룸(HIFU, RF, KTP 등) → 해당 기술을 사용하는 모든 장비에 연결 → `match_type: "technology"`

### 4.2 장비 조회 방법

```bash
# 전체 장비 목록
python papers/save.py --list-devices

# 키워드 검색
python papers/save.py --list-devices "HIFU"

# 기술 매칭 확인
python papers/save.py --match-technology "HIFU"
```

### 4.3 매칭 시 주의

- **직접 매칭**: 논문에 정확한 장비명/브랜드가 나올 때만
- **기술 매칭**: mechanism 필드 기반 자동 매칭. 사람 판단 불필요
- **미매칭**: 관련 장비가 DB에 없으면 `device_matches: []`로 두고, 어떤 장비가 필요한지 메모

## 5. 사전 검증 체크리스트

분석 전 반드시 확인:

### 5.1 학술 논문 여부
- [ ] 저자, 소속, 학술지명이 있는가?
- [ ] Abstract, Introduction, Methods, Results, Discussion 구조인가?
- [ ] DOI 또는 PMID가 있는가?
- **비학술 자료** (AI 정리물, 블로그, 개인 컴파일) → 분석하지 않고 사용자에게 알림

### 5.2 미용의료 관련성
- [ ] 피부 시술(레이저, 초음파, 고주파, 필러, 보톡스 등) 관련인가?
- [ ] 시술 원리/기전 연구도 관련으로 포함
- **비관련** (순수 치과학, 내과 등) → 사용자에게 알리고 분석 여부 확인

### 5.3 중복 체크
```bash
python papers/save.py --check-duplicate "10.xxxx/xxxxx"
python papers/save.py --check-duplicate "논문 제목"
```

## 6. DB 저장 절차

### 6.1 분석 결과 JSON 저장

분석 완료 후 JSON 파일로 저장:
```bash
# 파일로 저장 후
python papers/save.py paper_result.json
```

### 6.2 대량 처리 시 작업 순서

```
1. 폴더 내 PDF 목록 확인
2. 각 PDF에 대해:
   a. 학술 논문 검증
   b. 중복 체크
   c. 분석 → JSON 생성
   d. 장비 매칭
   e. DB 저장
3. 결과 리포트 출력:
   - 성공: N건
   - 중복 스킵: N건
   - 비학술 스킵: N건
   - 비관련 알림: N건 (파일명 나열)
   - 장비 미매칭: N건 (파일명 + 논문에서 언급된 장비 나열)
```

## 8. 잘못된 정보 발견 시

분석 중 아래 상황을 발견하면 **반드시 사용자에게 알림**:
- 파일명과 실제 내용이 다른 경우 (예: "HIFU 콜라겐" 파일이 실제로는 치과 논문)
- 비학술 자료 (AI 생성물, 개인 정리)
- 미용의료와 무관한 논문
- 동일 논문의 중복 파일

알림 형식:
```
⚠ [파일명 불일치] [기전] HIFU 콜라겐 변화 (2024).pdf
  - 파일명: HIFU 콜라겐 변화
  - 실제 내용: 치과 영역 초음파 영상 리뷰
  - 권장: 폴더에서 제거 또는 이동
```
